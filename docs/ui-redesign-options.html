MedTracker UI System Plan

Choose one visual language, then make the whole product obey it.

The product already has solid flows. The design problem is cohesion: the app currently mixes multiple surface systems, several typographic voices, and too many special-case shells across navigation, cards, forms, popovers, sheets, and modals. This plan treats the redesign as a system cleanup, not a random beautification pass.

Subagent Findings

  • Audit 1: design-system drift is strongest in surfaces, shadows, radii, and control chrome.
  • Audit 2: the app is readable, but not visually decisive; modals and navigation feel like separate products.
  • Conclusion: unify shells first, then page families, then premium brand moments.

Non-Negotiables

  • One modal sheet/dialog/popover contract
  • One control surface recipe for buttons, inputs, selects, comboboxes
  • One navigation shell across sidebar, top bar, bottom nav, and menus
  • One typography hierarchy with brand moments used sparingly

Direction Choice

Four coherent directions

A decision is required here. These options are intentionally distinct so the application stops drifting between moods. Pick one direction and the implementation plan below will lock the rest of the UI to it.

Clinical White + Sage

Calm, premium, and trustworthy. White surfaces stay crisp; sage carries the accent burden; gradients become rare and intentional.

Recommended

Character

This is the best match for healthcare without becoming cold. It gives MedTracker a cleaner, more expensive feel while keeping tasks obvious and stress-free.

Best For

  • Modals, forms, and administration flows
  • Inventory and medication detail pages
  • A calmer, cleaner app-wide shell

Editorial Wellness

Warm, tactile, and a little luxurious. More atmosphere, richer gradients, and stronger hero moments.

Expressive

Character

This pushes MedTracker toward a more branded and memorable identity, especially on profile, onboarding, and dashboard storytelling moments.

Risk

  • Can become decorative if applied to every utility screen
  • Needs strong restraint on alerts, tables, and inventory tools
  • Best used if brand personality matters as much as operational speed

Command Center

Sharper, more operational, and more decisive. Higher information density, stronger action hierarchy, cleaner status cues.

Operational

Character

This is the best direction if MedTracker should feel more like a clinical operations console than a wellness product. It improves scan speed and task confidence.

Best For

  • Medication actions and stock administration
  • Timeline, reports, and inventory dashboards
  • Dense mobile and tablet workflows

Soft Material System

Layered, translucent, and smooth. A system-first direction focused on consistency rather than brand drama.

Systematic

Character

This is the lowest-risk route if consistency is the top priority and you want fewer visual extremes.

Risk

  • Can drift back toward washed-out grey if contrast is not policed
  • Less memorable than the other three directions
  • Needs very disciplined typography to avoid feeling generic

Execution Plan

Rollout in three passes

Once a direction is chosen, the implementation should move in this order so the app becomes cohesive quickly without creating another layer of drift.

1

Foundation Pass

Lock typography, radii, shadows, borders, and the surface stack. Unify dialogs, sheets, popovers, dropdowns, selects, comboboxes, and control chrome before touching page-specific styling.

2

Shell Pass

Rebuild the app chrome so sidebar, mobile nav, top bar, profile menus, and flash/overlay behavior all feel like one product. This is where the app starts feeling intentional.

3

Page-Family Pass

Normalize the highest-traffic families: dashboard, medications, people, workflows/forms, and profile. Reserve premium or expressive moments for a few strategic hero surfaces only.

Recommended implementation order if you choose Clinical White + Sage: modal and floating surfaces, control primitives, navigation shell, dashboard and medications family, then profile/settings as the final polish pass.
