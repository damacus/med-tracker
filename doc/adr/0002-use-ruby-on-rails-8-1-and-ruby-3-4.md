# 2. Use Ruby on Rails 8.1 and Ruby 3.4

Date: 2026-04-13

## Status

Accepted

## Context

Need a modern, stable, and highly productive web framework for MedTracker.

## Decision

Use Ruby on Rails 8.1 and Ruby 3.4.

## Consequences

- Full access to modern Rails features (propshaft, solid queue/cache).
- Utilize modern Ruby language features (pattern matching, better performance).
- Consistent tooling through Ruby ecosystem standards.
